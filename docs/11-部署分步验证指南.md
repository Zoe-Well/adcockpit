# AdCockpit 部署分步验证指南

> 从 0 开始，每步验证通过再走下一步。  
> 核心原则：**不要加 `railway.toml`，让 Nixpacks 自动检测。**

---

## 第 1 步：确认代码状态

**本地确认**：

```bash
# 1. 确保不在 Git 里留 railway.toml（如果有就删掉）
ls railway.toml 2>/dev/null && echo "删除它" || echo "OK，没有"

# 2. 确认推送成功
git push origin main

# 3. 确认 GitHub 上能看到最新代码
# 浏览器打开 https://github.com/Zoe-Well/adcockpit
```

**验证通过标志**：GitHub 上代码是最新的，根目录没有 `railway.toml`。

---

## 第 2 步：Railway 部署后端

### 2.1 删除旧服务，新建

1. Railway → 打开你之前那个项目
2. 左边 Service 卡片 → 右上角 `⋯` → **Delete Service**
3. 点 **New Service** → **Deploy from GitHub Repo**
4. 选 `Zoe-Well/adcockpit`

> 什么都不用配，Nixpacks 会自动：检测 Python → 读 `requirements.txt` → 设启动命令 → 启动。  
> 观察 Build Log，看到 `Nixpacks build succeeded` 和 `Application startup complete` 就是部署成功了。

### 2.2 加环境变量

Settings → Variables → 添加：

| Key | Value |
|-----|-------|
| `DEEPSEEK_API_KEY` | `sk-你的DeepSeek密钥` |
| `DEEPSEEK_BASE_URL` | `https://api.deepseek.com/v1` |
| `API_SECRET_KEY` | `adcockpit-demo-2026`（自定义） |

加完后点 **Deploy** 生效。

### 2.3 开启公网访问

Settings → Networking：

1. **Enable Public Networking**
2. TCP Proxy → 添加端口 `8080`
3. 点 **Generate Domain**
4. 得到一个 `xxx.up.railway.app` 地址

### 2.4 验证

```bash
# 浏览器打开
https://你的域.up.railway.app/health
```

**能访问的标志**：看到 `{"status":"ok","version":"2.0.0"}`。

**如果 404**：确认 Runtime Logs 里 `Application startup complete` 且没有后续报错。如果有——发日志。

---

## 第 3 步：Vercel 部署前端 + 连通后端

### 3.1 确认前端部署

如果之前已经部署过，直接用原来的 Vercel 项目。没有就新建：

1. [vercel.com](https://vercel.com) → Add New → Project → 选 repo
2. Settings → General → Root Directory → 填 `frontend` → Save

### 3.2 配环境变量

Settings → Environment Variables → 添加：

| Key | Value |
|-----|-------|
| `VITE_API_URL` | `https://你的域.up.railway.app`（第 2 步 Railway 给的地址） |
| `VITE_API_KEY` | `adcockpit-demo-2026`（和 Railway 设的一致） |

改完后点顶部 **Deployments** → 最新的那条 → Redeploy。

### 3.3 验证前后端连通

浏览器打开 Vercel 给的 `.vercel.app` 地址 → 点 **"优化抖音最近7天的投放ROI"** → 应该能看到参数面板和 Trace 动画。

**连通的标志**：Trace 动画能跑完，出现审批卡片。  
**连不通的标志**：页面显示"系统暂时无法响应"——说明 `VITE_API_URL` 没配对或 Railway 后端有问题。

---

## 第 4 步：阿里云域名绑定（国内访问）

> 前 3 步验证通过后再做这一步。

### 4.1 前端绑定 Vercel

**Vercel 端**：Settings → Domains → 添加 `adcockpit.zoewell.cn`

Vercel 提示 DNS 记录，去[阿里云 DNS](https://dns.console.aliyun.com)：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| CNAME | `adcockpit` | `cname.vercel-dns.com` |

**验证**：等 5-10 分钟，访问 `https://adcockpit.zoewell.cn` → 看到 AdCockpit 界面。

### 4.2 后端绑定 Railway

**Railway 端**：Settings → Custom Domain → 添加 `api.zoewell.cn` → Add

弹窗显示 **DNS Target** 值 → 复制。去阿里云：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| CNAME | `api` | Railway 弹窗里的 DNS Target |

> 状态列黄色 = SSL 证书申请中，5-15 分钟变绿 ✅。

**验证**：`https://api.zoewell.cn/health` → `{"status":"ok"}`。

### 4.3 更新环境变量

Vercel → Settings → Environment Variables → 把 `VITE_API_URL` 改成 `https://api.zoewell.cn`。

改完点 **Redeploy**。

### 4.4 最终验证

用手机 4G 打开 `https://adcockpit.zoewell.cn` → 完整流程走通：

```
快捷提示 → 参数面板 → 开始优化 → Trace 动画 → 审批卡片 → 确认执行 → 完成
```

## 踩坑速查

| 现象 | 原因 | 解决 |
|------|------|------|
| `railpack process exited` | 缺 `requirements.txt` | 根目录必须有 `requirements.txt` |
| Railway 404 "Application not found" | Nixpacks 启动命令不匹配 | **删掉 `railway.toml`**，让 Nixpacks 自动检测 |
| 前端"系统暂时无法响应" | `VITE_API_URL` 不对 | 检查 Vercel 环境变量 |
| 前端 API 401 | `VITE_API_KEY` 和 `API_SECRET_KEY` 不一致 | 两边用同一个值 |
| 域名黄色不变绿 | SSL 证书在签发中 | 等 15 分钟，超时删掉域重新加 |
| 域能解析但 404 | Railway 后端本身有问题 | 先验证 `up.railway.app` 原始域可用 |
