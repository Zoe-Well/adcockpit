## P0 — 阻断（必须立即修复）

### 1. API 失败后显示空的审批卡片

**位置**: [App.tsx:172](../frontend/src/App.tsx#L172)、[App.tsx:213](../frontend/src/App.tsx#L213)、[App.tsx:358](../frontend/src/App.tsx#L358)

**现象**: `runOptimize()` / `runContent()` / `runCreate()` 的 catch 块执行 `setApproval(true)`，导致 API 调用失败后向用户展示了一个**空的审批对话框**——没有数据、没有操作项，只有一个无内容的确认框。

**影响**: 用户看到空白审批卡片，完全不知道发生了什么，无法进行任何操作。

**修复方向**:
```ts
// 不要 setApproval(true)，改为设置错误状态
catch(e) {
  clearInterval(thinkingIv)
  setAgentSteps([])                                          // 清空调试步骤
  setMsgs(p => [...p, {role:'agent', content:'⚠️ 执行遇到问题，请稍后重试。'}])
  // 不设置 approval
}
```

### 2. 直播/数据/诊断三个标签页完全是空壳

**位置**: [App.tsx:657-679](../frontend/src/App.tsx#L657-L679)

**现象**: 点击"直播监控""数据分析""故障诊断"标签页，仪表盘显示的是投放优化的 ROI 数据（走了 else 分支）。没有任何与当前标签相关的功能或数据。

**用户视角**: 用户点"直播监控"，看到的是 ROI 柱状图和投放计划表格——完全不知道这和直播有什么关系。

**修复方向**: 短期方案——为未实现的标签添加"即将上线"占位页；长期方案——实现真实业务逻辑。

## P1 — 主要（强烈建议本轮修复）

### 3. 新用户首次使用零引导

**位置**: [App.tsx:61](../frontend/src/App.tsx#L61)

**现象**: 新用户只看到一条 AI 欢迎消息，没有任何引导式教程、示例查询快捷方式、或新手任务。用户必须自己摸索正确的输入方式。

**修复方向**: 欢迎消息后追加 3-4 个可点击的快捷提示词：
```
"试试这些："
[优化抖音最近7天ROI] [生成夏季促销带货脚本] [新建一个广告投放计划]
```

---

### 4. 后端断连时暴露技术错误给非技术用户

**位置**: [App.tsx:125](../frontend/src/App.tsx#L125)

**现象**: 用户看到：`后端未连接。先启动: uvicorn backend.app.api.main:app --port 8000`

这是给开发者看的命令行，普通用户（营销人员）看到会完全困惑——"uvicorn 是什么？端口是什么？"

**修复方向**: 改为用户友好提示：
```
"系统暂时无法响应，请检查网络连接或稍后重试。[重试]"
```

---

### 5. 切换标签页会丢弃进行中的工作流

**位置**: [App.tsx:443](../frontend/src/App.tsx#L443)

**现象**: 用户在"投放优化→等待审批"状态下点击其他标签页，所有 trace 步骤、审批卡片、参数面板被瞬间清空，无法恢复。

**修复方向**: 切换标签时检测是否有进行中的工作流（`approval === true`），弹确认框："当前有未完成的操作，切换标签将丢失进度。确定离开吗？"

---

### 6. "English / 中文"按钮完全无效

**位置**: [App.tsx:454](../frontend/src/App.tsx#L454)

**现象**: 侧边栏底部的语言切换按钮没有 `onClick` 处理函数。点击无任何反应。

**修复方向**: 要么接 `react-i18next` 实现真正的国际化（中/英双语），要么暂时移除该按钮。一个不起作用的按钮比没有按钮更损害用户信任。

---

### 7. 优化执行后看不到"之前 vs 之后"对比

**位置**: [App.tsx:271-316](../frontend/src/App.tsx#L271-L316)

**现象**: 确认执行优化 → 仪表盘刷新 → 用户看到新数据，但完全不知道变了什么。ROI 从多少变到多少？节省了多少钱？没有标注。

**修复方向**: 执行前快照 `plans` 数据 → 执行后对比 → Dashboard 卡片显示变化值：
```
ROI: 1.87 → 2.12 (+13.4% ↑)
日节省: ¥6,400
```

---

### 8. 内容生成后无法在 UI 中查看完整脚本

**位置**: [App.tsx:259](../frontend/src/App.tsx#L259)

**现象**: 生成成功后，脚本被截断为 80 字符 + "..."，用户无法在应用内看到完整内容。

**修复方向**: 聊天消息中每条脚本用可展开区块显示，或 Dashboard 增加"脚本预览"面板。

---

### 9. 三栏布局在笔记本屏幕上不可用

**位置**: [App.tsx:461-682](../frontend/src/App.tsx#L461-L682)

**现象**: 三个面板的 `min-w-[360px]` + `min-w-[360px]` + `min-w-[440px]` = 最小需要 **1,160px** 屏幕宽度。在 13-14 寸笔记本（通常 1024-1440px）上严重挤压或溢出。

**修复方向**: 使用 Tailwind 响应式断点：
- `lg`（1024px）：对话+追踪合并为标签切换
- `md`（768px）：单栏，底部 Tab 栏切换三个面板

---

### 10. 取消操作后无法撤销

**位置**: [App.tsx:481](../frontend/src/App.tsx#L481)

**现象**: 用户点击审批卡片的"取消"→ 审批卡片消失 + 聊天框提示"操作已取消"。如果是不小心点的取消，没有恢复途径。

**修复方向**: 取消后显示一条带"撤销"按钮的提示（5 秒内可点击恢复）：
```
"操作已取消  [撤销]"
```

---

### 11. 标题栏日期硬编码

**位置**: [App.tsx:435](../frontend/src/App.tsx#L435)

**现象**: 标题栏显示 `2026-06-14`——这是写死在代码里的字符串，从不会变化。

**修复方向**: 改为 `new Date().toLocaleDateString('zh-CN')` 实时显示当天日期。

---

### 12. loadPlans 加载时无任何指示

**位置**: [App.tsx:91-96](../frontend/src/App.tsx#L91-L96)

**现象**: 页面首次加载和每次操作后，Dashboard 区域（占屏幕 42%）静默刷新——没有骨架屏、没有 spinner、没有"加载中"提示。慢网络下用户可能以为系统卡死了。

**修复方向**: 加载期间显示骨架屏或至少一个 spinner overlay。

---

### 13. 优化参数面板的输入框不是受控组件

**位置**: [App.tsx:516-517](../frontend/src/App.tsx#L516-L517)

**现象**: 参数面板中"出价调整 %"和"预算调整 %"使用 `defaultValue`（非受控），无论用户怎么改，执行时始终用硬编码的 `-10` 和 `-20`。用户以为可以自定义，实际不行。

**修复方向**: 改为受控组件（`value` + `onChange` + state），让参数真正对执行生效。

## P2 — 值得优化（可在后续迭代中处理）

### 14. 初始状态填充了假数据

用户首次打开就显示 10 条模拟计划，误以为系统已有真实数据。应显示空状态 + "创建第一个计划"引导。

### 15. 思考动画与实际后端进度无关

`runOptimize()` 的 thinking 文本是硬编码数组以固定间隔播放——后端 3 秒返回则动画戛然而止，后端 15 秒返回则动画播完留下空白等待。应接入 WebSocket/SSE 实时事件。

### 16. "预览脚本"按钮命名误导

`runContent()` 的按钮叫"预览脚本"，但点击后触发完整 AI 生成流程（拉取素材→分析→生成→审批）。应改名"生成脚本"或拆分为真正的预览 + 确认生成两步。

### 17. 内容生产脚本截断

生成成功后显示 `s.substring(0,80)+'...'`，用户看不到完整脚本。应支持展开查看。

### 18. 执行后无法回滚

确认执行调价/创建后没有"撤销"按钮。应保存操作前快照，支持短期回滚。

### 19. 仪表盘不聚合用户创建的计划

`创建的投放计划` 存在 `createdList` state 中，但 Dashboard 的 `在投计划` 只显示 `plans.length`（来自 mock 数据）。用户创建的计

划被排除在外。

### 20. 仪表盘无"数据更新时间"显示

数据刷新后，用户不知道数据是几秒前还是几分钟前更新的。应添加 `上次更新: HH:MM:SS` 时间戳。

- **[bug.md](bug.md)**: 记录已发现和已修复的**技术 Bug**（代码逻辑错误、状态管理问题）
