# AdCockpit v1.0 核心知识点梳理

> 面向数字营销 AI Agent 架构师岗位，结合项目实际实现。

---

## 一、架构模式

### 当前架构（v1.0）：Streamlit 直连

```
Streamlit (8501) → 直接调用 tools/mock_functions.py + tools/feishu_client.py
```

单进程，`streamlit run ui/app.py` 即可运行。保留 `api/` 三层架构可选。

---

## 二、LangGraph Agent 编排

- StateGraph + 条件边 + MemorySaver
- 8 个专业 Agent 节点
- HITL：`interrupt()` + `Command(resume=...)`
- 并行：`ThreadPoolExecutor` 在节点内并行调用多平台 API

---

## 三、意图识别

```
输入 → 对话检测 → 业务检测 → 场景路由 → 参数面板
```

- 疑问句优先对话
- 动作词+业务目标 → 业务请求
- 单一关键词 → 引导回复

---

## 四、流式动画引擎

- `_trace_animating` + `_trace_step` 驱动
- 每节点 3 帧，每帧 0.15s
- 文字逐字打印 + 流程图逐节点点亮
- 动画完成后停在确认步骤等待用户操作

---

## 五、六标签系统

| 标签 | 流程 |
|------|------|
| 🎯 优化 | 参数→动画→确认→执行 |
| 📊 投放 | 表单→动画→确认→上线 |
| 🎬 内容 | 参数→预览→动画→确认→飞书 |
| 🛒 电商 | 仪表盘展示 |
| 📈 数据 | 仪表盘展示 |
| 🔧 诊断 | 仪表盘展示 |

---

## 六、飞书 API 集成

- `GET /auth/v3/tenant_access_token/internal` → token
- `POST /docx/v1/documents` → 创建文档
- `POST /docx/v1/documents/{id}/blocks/{id}/children` → 写入内容
- 未配置凭证时自动 Mock 降级

---

## 七、数字营销核心指标

ROI > 2.0 | CPA < ¥50 | CTR > 2% | CVR > 2%

Dashboard 实时展示，低于阈值自动高亮。

---

## 八、设计原则

1. 确认后执行 — 数据修改前必须用户确认
2. 可观测性 — 每一步在 Trace Board 可视化
3. 单一职责 — 每个 Agent 只做一件事
4. Mock 异常模拟 — 10% 业务 + 5% 网络异常
