## 一、本地开发环境搭建（从零开始）

### 1.1 需要的工具

| 工具 | 用途 | 安装方式 |
|------|------|---------|
| **Python 3.11+** | 后端运行环境 | [python.org](https://python.org) 下载安装 |
| **Node.js 20+** | 前端运行环境 | [nodejs.org](https://nodejs.org) 下载安装 |
| **Git** | 代码版本管理 | [git-scm.com](https://git-scm.com) 下载安装 |
| **VS Code**（推荐）| 代码编辑器 | [code.visualstudio.com](https://code.visualstudio.com) |

### 1.2 验证安装

打开终端（Win+R → 输入 `cmd` 或 `powershell`），运行：

```bash
python --version    # 应显示 Python 3.11.x 或更高
node --version      # 应显示 v20.x 或更高
npm --version       # 应显示 10.x 或更高
git --version       # 应显示 2.x 或更高
```

### 1.3 克隆项目

```bash
git clone <项目地址> adcockpit
cd adcockpit
```

### 1.4 后端环境

```bash
# 1. 创建虚拟环境（隔离项目依赖，不污染系统 Python）
python -m venv venv

# 2. 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# 3. 安装依赖
pip install -r backend/requirements.txt
```

**什么是虚拟环境？** 一个隔离的 Python 运行环境。项目 A 用 FastAPI 0.1，项目 B 用 FastAPI 0.2——它们互不干扰。类似给每个项目一个独立的工具箱。

### 1.5 前端环境

```bash
cd frontend

# 安装依赖（根据 package.json 自动下载所有需要的包）
npm install
```

## 二、启动项目（开发模式）

开发时需要**同时启动前端和后端**，开两个终端窗口。

### 2.1 启动后端（终端 1）

```bash
# 确保在项目根目录，虚拟环境已激活
cd backend
uvicorn app.api.main:app --port 8000 --reload
```

```
uvicorn app.api.main:app  ← 模块路径:应用实例
           --port 8000     ← 监听端口
           --reload        ← 代码改动时自动重启（开发专用）
```

启动成功后：
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

验证：浏览器打开 `http://localhost:8000/health` → 应显示 `{"status":"ok"}`  
API 文档：打开 `http://localhost:8000/docs` → 交互式 Swagger 文档

### 2.2 启动前端（终端 2）

```bash
cd frontend
npm run dev
```

启动成功后：
```
VITE v5.x  ready in xxx ms
➜  Local:   http://localhost:5173/
```

浏览器打开 `http://localhost:5173` 即可看到 AdCockpit 界面。

### 2.3 开发模式热重载

- **后端**：`--reload` 确保改 Python 代码后自动重启
- **前端**：Vite 默认 HMR（Hot Module Replacement），改 React 代码后浏览器**不刷新**就更新

## 三、环境变量配置

### 3.1 什么是环境变量？

**不要把密码/密钥写在代码里。** 用环境变量——运行时从配置文件读取。

```
代码里：DEEPSEEK_API_KEY = os.getenv("DEEPSEEK_API_KEY")
配置里：DEEPSEEK_API_KEY=sk-xxxxxxxx
```

这样代码可以公开（不含密码），配置文件只保存在本地。

### 3.2 AdCockpit 需要的环境变量

创建 `backend/.env` 文件：

```bash
# DeepSeek LLM API（必需）
# 获取地址：https://platform.deepseek.com/api_keys
DEEPSEEK_API_KEY=sk-你的密钥
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1

# 飞书文档 API（可选，内容发布功能需要）
# 获取地址：https://open.feishu.cn/app
FEISHU_APP_ID=cli_你的应用ID
FEISHU_APP_SECRET=你的应用密钥
FEISHU_FOLDER_TOKEN=飞书文件夹Token（可选）

# Redis（可选，异步任务/Celery 需要）
REDIS_URL=redis://localhost:6379

# Chroma（可选，RAG 向量检索需要）
CHROMA_HOST=localhost
CHROMA_PORT=8001
```

### 3.3 密钥获取指南

**DeepSeek API Key**：
1. 访问 https://platform.deepseek.com
2. 注册账号 → 充值（最低 ¥1 即可开始）
3. API Keys → 创建新 Key → 复制

**飞书应用凭证**（内容发布功能需要）：
1. 访问 https://open.feishu.cn/app
2. 创建企业自建应用 → 获取 App ID + App Secret
3. 权限管理 → 添加 `docx:document` 权限
4. 安全设置 → 添加后端服务器 IP

## 四、Docker 容器化部署

### 4.1 什么是 Docker？

**Docker 把应用和它的所有依赖打包成一个"容器"**，在任何机器上运行结果一致。

```
传统部署：
  "在我机器上能跑啊" → 服务器环境不同 → 各种报错

Docker 部署：
  开发机容器能跑 → 服务器容器也能跑 → 完全一致
```

**核心概念**：

| 概念 | 类比 | 说明 |
|------|------|------|
| **Image（镜像）** | 模具 | 包含代码 + 依赖 + 系统库的只读模板 |
| **Container（容器）** | 用模具做出的零件 | 镜像的运行实例，隔离运行 |
| **Dockerfile** | 模具设计图 | 定义如何构建镜像 |
| **docker-compose.yml** | 装配说明书 | 定义多个容器如何协同工作 |

### 4.2 Dockerfile（后端）

```dockerfile
# 基于官方 Python 镜像
FROM python:3.11-slim

# 设置工作目录
WORKDIR /app

# 先复制依赖文件（利用 Docker 缓存层）
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uvicorn", "backend.app.api.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**关键说明**：
- `FROM python:3.11-slim`：基于轻量 Python 镜像（`slim` = 精简版，体积小）
- `COPY requirements.txt .` 和 `COPY . .` 分开写：如果只改代码不改依赖，Docker 可以重用缓存层，构建更快
- `--host 0.0.0.0`：容器的 localhost 不等于宿主机的 localhost，必须用 `0.0.0.0` 绑定所有网络接口

### 4.3 Dockerfile（前端）

```dockerfile
# 构建阶段
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json .
RUN npm ci
COPY . .
RUN npm run build          # 构建生产版本 → dist/

# 运行阶段（Nginx 静态文件服务）
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

**为什么分两个阶段（Multi-stage Build）？**
- 构建阶段：需要 Node.js + npm + 源码 → 编译成静态文件
- 运行阶段：只需要 Nginx + 编译好的文件 → 镜像体积从 ~1GB 降到 ~50MB

### 4.4 docker-compose.yml（一键启动全部服务）

```yaml
version: '3.8'
services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8000:8000"          # 宿主机:容器
    env_file:
      - ./backend/.env
    depends_on:
      - redis
    restart: unless-stopped

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    restart: unless-stopped

  # chroma（可选——RAG 功能需要）
  chroma:
    image: chromadb/chroma:latest
    ports:
      - "8001:8001"
    restart: unless-stopped
```

**启动全部服务**：
```bash
docker-compose up -d     # -d = 后台运行
docker-compose ps         # 查看运行状态
docker-compose logs -f    # 查看日志
docker-compose down       # 停止并删除容器
```

### 4.5 Docker 常用命令

```bash
# 构建镜像
docker build -t adcockpit-backend .

# 运行容器
docker run -p 8000:8000 --env-file .env adcockpit-backend

# 查看运行中的容器
docker ps

# 查看容器日志
docker logs <容器ID> -f

# 进入容器内部调试
docker exec -it <容器ID> bash

# 停止容器
docker stop <容器ID>

# 删除容器
docker rm <容器ID>

# 清理无用镜像（释放磁盘空间）
docker image prune -a
```

## 五、生产环境部署

### 5.1 部署架构

```
互联网
  │
  ▼
Nginx（反向代理 + 静态文件）
  ├── /              → 前端静态文件（React 编译产物）
  ├── /api/*         → 后端 FastAPI (:8000)
  └── /ws/*          → 后端 WebSocket (:8000)
```

**为什么需要 Nginx？**
1. **反向代理**：隐藏后端真实地址，统一入口
2. **静态文件服务**：Nginx 处理静态文件比 Python 快 100 倍
3. **HTTPS 终止**：Nginx 处理 SSL 证书，后端只需处理 HTTP
4. **负载均衡**：流量大时分发到多个后端实例

### 5.2 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/adcockpit;
        try_files $uri /index.html;    # SPA 路由支持
    }

    # API 代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket 代理
    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 5.3 云服务器部署流程（以阿里云为例）

```
步骤 1：购买服务器
  - 最低配置：2核 4GB（约 ¥50-100/月）
  - 操作系统：Ubuntu 22.04
  - 开放端口：80（HTTP）、443（HTTPS）、22（SSH）

步骤 2：SSH 登录
  ssh root@你的服务器IP

步骤 3：安装必要工具
  apt update
  apt install -y python3.11 python3.11-venv nginx git curl

步骤 4：安装 Node.js
  curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
  apt install -y nodejs

步骤 5：克隆项目
  git clone <项目地址> /opt/adcockpit
  cd /opt/adcockpit

步骤 6：配置环境变量
  cp .env.example backend/.env
  nano backend/.env     # 填入真实的 API Key

步骤 7：构建前端
  cd frontend
  npm ci && npm run build
  cp -r dist/* /var/www/adcockpit/

步骤 8：配置后端为系统服务（自动重启）
  nano /etc/systemd/system/adcockpit.service
```

`adcockpit.service` 内容：

```ini
[Unit]
Description=AdCockpit Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/adcockpit
EnvironmentFile=/opt/adcockpit/backend/.env
ExecStart=/opt/adcockpit/venv/bin/uvicorn backend.app.api.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
systemctl daemon-reload
systemctl enable adcockpit
systemctl start adcockpit
systemctl status adcockpit   # 查看状态

# 配置 Nginx
nano /etc/nginx/sites-available/adcockpit
ln -s /etc/nginx/sites-available/adcockpit /etc/nginx/sites-enabled/
nginx -t                    # 测试配置
systemctl reload nginx      # 重载配置

# 配置防火墙
ufw allow 80
ufw allow 443
ufw allow 22
ufw enable
```

## 六、常见问题排查

### 6.1 后端启动失败

```bash
# 错误：ModuleNotFoundError: No module named 'fastapi'
# 原因：依赖未安装
pip install -r backend/requirements.txt

# 错误：Address already in use
# 原因：端口 8000 已被占用
lsof -i :8000           # 查看谁占用了端口
kill -9 <PID>           # 杀掉占用的进程

# 错误：uvicorn: command not found
# 原因：虚拟环境未激活或 uvicorn 未安装
venv\Scripts\activate   # Windows
pip install uvicorn
```

### 6.2 前端启动失败

```bash
# 错误：npm ERR! code ERESOLVE
# 原因：依赖版本冲突
rm -rf node_modules package-lock.json
npm install

# 错误：vite: command not found
# 原因：依赖未安装
npm install

# 错误：端口 5173 被占用
# 修改 vite.config.ts 或关闭占用进程
npx kill-port 5173
```

### 6.3 LLM 连接失败

```
现象：[LLM RETRY 1/3] error=Connection error
确认：curl https://api.deepseek.com/v1/models -H "Authorization: Bearer sk-xxx"
```

常见原因：
1. **API Key 错误**：检查 `backend/.env` 中的 key 是否正确
2. **网络代理**：公司网络可能需要配置代理 → 设置 `HTTPS_PROXY` 环境变量
3. **Key 余额不足**：登录 DeepSeek 控制台查看余额
4. **防火墙**：确认能访问 `api.deepseek.com:443`

### 6.4 前端请求后端失败

```
现象：浏览器控制台显示 net::ERR_CONNECTION_REFUSED
原因：后端未启动，或 CORS 配置错误
```

- 确认后端已启动：`curl http://localhost:8000/health`
- 确认 CORS 允许前端地址（`main.py` 中 `allow_origins`）

### 6.5 .env 文件不生效

```bash
# 查看 config.py 加载的 .env 路径
python -c "from backend.app.core.config import _ENV_FILE; print(_ENV_FILE)"

# 确认 .env 文件在该路径存在
ls -la backend/.env
```
