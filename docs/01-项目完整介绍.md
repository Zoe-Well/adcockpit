# AdCockpit v1.0 — AI 数字营销对话驾驶舱 · 项目完整介绍

> **一句话定位**：基于 LangGraph Multi-Agent 编排的全链路数字营销 AI 系统，
> 以三栏对话驾驶舱形态，通过自然语言驱动 Agent 协作，
> 覆盖广告投放、内容生产、电商闭环、数据分析四大核心业务。
> **95% 代码由 AI 生成，2 天内完成从概念到可运行 Demo。**

---

## 一、项目概述

AdCockpit 是一个面向数字营销服务商的 AI Agent 系统，解决"多平台数据孤岛、
决策-执行链路断裂、内容-投放-电商割裂"三大行业痛点。

与传统的 Chatbot 不同，AdCockpit 采用**三栏对话驾驶舱**架构：

```
┌──────────────┬──────────────────┬──────────────┐
│  左侧 32%    │    中间 26%       │  右侧 42%    │
│  对话面板    │  Agent 编排与     │  数据洞察     │
│  Chat Panel │  执行追踪         │  仪表盘       │
│             │  Trace Board     │  Dashboard   │
│  ·自然语言  │  ·流程追踪器      │  ·指标卡片    │
│  ·参数面板  │  ·步骤卡片+打字   │  ·柱状图      │
│  ·确认卡片  │  ·确认/取消按钮   │  ·分页表格    │
│  ·多轮对话  │                  │  ·报告预览    │
└──────────────┴──────────────────┴──────────────┘
```

## 三、技术架构

### 3.1 当前架构（v1.0）

```
Streamlit (8501)
  ├── 直接调用 tools/mock_functions.py（24个Mock函数）
  ├── 直接调用 tools/feishu_client.py（飞书文档API）
  └── 流式动画引擎（_trace_animating + _trace_step）
```

单进程架构，无需独立后端。优化、内容生产、飞书发布全部在 Streamlit 内完成。

### 3.2 保留的三层架构（可选）

```
Streamlit ──HTTP──→ FastAPI (8000) ──→ Mock/Feishu
```

`api/` 目录保留完整实现，内容生产可通过 FastAPI 调用飞书发布。

### 3.3 核心技术栈

| 层次 | 技术 | 版本 |
|------|------|------|
| 前端 | Streamlit | ≥1.28 |
| Agent 框架 | LangGraph | 1.0.10 (StateGraph + MemorySaver) |
| LLM 接口 | LangChain + langchain-openai | 1.2.3 / 1.0.1 |
| 后端 | FastAPI (可选) | 0.136.3 |
| 飞书 | Feishu Open API (docx) | v1 |
| 数据校验 | Pydantic | ≥2.0 |

## 四、核心功能详解

### 4.1 投放优化流程

```
输入需求 → 理解意图 → 参数面板（可调）→ 🚀开始优化
→ 流式动画(7步+打字效果) → 审批卡片 → 确认执行
→ 实际修改出价/预算 → 仪表盘数据更新 → 飞书报告
```

**参数面板**：投放平台、回溯天数、计划数量、ROI 阈值、出价/预算调整比例、风险等级——全部可调，AI 给默认值。

**流式动画**：中间面板逐节点点亮 + 文字逐字打印，模拟 Agent 思考过程（约 3 秒完成）。

**确认执行后才真正修改数据**，点击取消不产生任何影响。

### 4.2 内容生产流程

```
输入需求 → 参数面板 → 🔍预览脚本 → 流式动画 → 确认发布 → 飞书文档
```

**预览脚本**：先在对话区展示生成内容，用户可以检查和取消。

**飞书发布**：确认后通过飞书 API 创建真实文档（配置凭证后），或生成 Demo 链接。

### 4.3 投放计划创建

```
📊投放标签 → 🚀新建表单 → 填平台/预算/出价/定向
→ 提交 → 流式动画 → 确认执行 → 计划正式上线
```

## 五、意图识别

系统通过关键词匹配识别用户输入属于哪种意图：

| 输入类型 | 处理方式 |
|---------|---------|
| 对话/问候（"你好"、"你是什么"） | 友好文字回复 |
| 业务指令（"检查ROI"、"分析素材"） | 弹出参数面板 |
| 模糊输入 | 引导回复，列出能力范围 |

识别规则：必须同时包含**动作动词 + 业务目标**才算业务请求，单一关键词不触发。
