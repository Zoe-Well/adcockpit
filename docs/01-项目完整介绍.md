## 一、项目概述

AdCockpit 是一个面向数字营销服务商的 AI Agent 系统，解决三大行业痛点：
- 多平台数据孤岛（抖音 + 腾讯广告）
- 决策-执行链路断裂（LLM 分析 → 策略 → 人工审批 → 执行）
- 内容-投放-电商割裂（一个系统打通）

与传统的 Chatbot 不同，AdCockpit 采用**三栏对话驾驶舱**架构——

```
┌──────────────┬──────────────────┬──────────────┐
│  左侧 30%    │    中间 28%       │  右侧 42%    │
│  对话面板    │  Agent 编排追踪    │  数据洞察     │
│  Chat Panel │  Trace Board     │  仪表盘       │
│             │                  │  Dashboard   │
│  ·快捷提示  │  ·流程管线        │  ·指标卡片    │
│  ·自然语言  │  ·步骤卡片+丰富   │  ·ROI柱状图   │
│  ·参数面板  │   思考过程展示    │  ·before/after│
│  ·确认卡片  │  ·展开/收起详情   │  ·分页表格    │
│  ·撤销提示  │  ·LLM精准标签    │  ·更新时间戳  │
│             │  ·确认/取消按钮   │  ·场景专属    │
└──────────────┴──────────────────┴──────────────┘
```

## 三、技术架构

### 3.1 v2.0 架构

```
React 18 + Tailwind + shadcn/ui (:5173)
        │  X-API-Key 认证（生产环境）
        ▼
FastAPI (:8000)
  ├── API Key 中间件（/api/* 保护，本地可关闭）
  ├── POST /api/intent/classify   → DeepSeek LLM（3次重试 → keyword fallback）
  ├── POST /api/agent/optimize    → LangGraph 7 节点投放优化编排
  ├── POST /api/agent/content     → 6 节点内容生产编排
  ├── POST /api/agent/create      → 4 节点计划创建编排
  ├── POST /api/campaigns/*       → 投放计划 CRUD + JSON 持久化
  ├── POST /api/content/*         → 脚本生成（DeepSeek）+ 飞书发布
  ├── GET  /api/campaigns/all     → Dashboard 数据加载
  └── GET  /ws/{session_id}       → WebSocket 实时推送（端点就绪）
        │
        ▼
Mock Functions (24) + Feishu Open API + Local JSON (backend/data/)
```

### 3.2 部署架构

```
  VITE_API_URL 构建时注入           API_SECRET_KEY 运行时读取
```

## 四、核心功能详解

### 4.1 投放优化（US1，100%）

```
用户输入 "优化投放ROI"（或点击快捷提示按钮 💬）
  → LLM 意图识别 → tab 自动跳转投放优化
  → 参数面板（可调时间范围/ROI阈值/出价%/预算%——全部受控生效）
  → 🚀 开始优化 → 思考动画 → LangGraph 7 节点逐条展示
  → 审批卡片（异常逐条 + 风险等级 HIGH/MEDIUM/LOW）
  → 确认执行 → execute ◉转圈 → API 调价 → execute ✓ + report ✓
  → Dashboard 刷新 + before/after ROI 变化箭头（↑↓）
  → 飞书报告
```

**Trace Board 输出**：Supervisor（置信度+执行计划列表）、
Data（逐条计划 ID/ROI/消耗/出价详细数据）、
Analysis（🔴🟡异常标注 + 占比）、
Strategy（逐条方案 + 预估日节省金额）。
超过 3 行自动折叠+渐变遮罩，点击展开。

**运行中→完成流程**：点击确认后 report 节点先显示 ◉ 转圈（"正在汇总执行结果…"），API 返回后才变 ✓ 绿勾。API 失败显示 ✕ + 友好错误消息。取消时显示"操作已撤销"提示。切换标签时如有进行中的审批流，弹出确认框保护进度。

### 4.2 内容生产（US2，100%）

```
用户输入 "生成带货脚本"
  → LLM 意图识别 → tab 跳转内容生产
  → 参数面板（平台/模板/数量）
  → 生成脚本 → 6 节点逐步展示
  → 审批卡片（动态显示模板名 + 数量）
  → 确认发布 → execute ◉ → API 生成 + 飞书发布 → execute ✓ + report ✓
  → 聊天框展示完整脚本（非截断），Dashboard 更新生成记录
```

**Trace Board 展示**：素材 CTR 对比（高CTR 3条 5.7% vs 低CTR 3条 1.5%）、
爆款特征 4 项（含定量提升数据）、脚本结构 4 段拆解（钩子→卖点→优惠→CTA）。

### 4.3 广告投放计划创建（US6，95%）

```
点击广告投放标签 → 填写表单（平台/名称/预算/出价）
  → 提交 → Agent 4 节点编排
  → 审批卡片（预估日曝光 8K-12K + 费用提醒）
  → 确认投放 → execute ◉ → API 创建 → execute ✓ + report ✓
  → Dashboard 表格更新 + 创建记录列表 + 计划正式上线
```

## 十一、项目亮点

**Multi-Agent 编排**：基于 LangGraph StateGraph 构建 8 个专业 Agent 节点（Supervisor / Data / Analysis / Strategy / Content / Ecommerce / Execute / Report），其中 6 个接入主编排图实现投放优化全链路自动化。支持条件路由分支与 Human-in-the-Loop 审批中断——高风险调价操作自动暂停在 Execute 节点，等待人工确认后才调用平台 API 执行出价和预算修改。

**业务覆盖**：打通广告投放优化与内容生产两大核心域（全链路完成），覆盖广告投放计划创建（95% 完成），预留电商场控/数据分析/故障诊断三域骨架并标注开发进度。通过飞书文档 Open API 实现优化报告与带货脚本的自动发布，基于 Mock 函数模拟抖音和腾讯广告双平台广告系统，构建 24 个业务工具（内置 10% 业务异常 + 5% 网络异常，测试系统鲁棒性）。

**思考过程可视化**：Trace Board 中每个 Agent 节点展示 3-10 行详细推理输出——Supervisor 包含置信度与执行计划列表，Data 包含逐条计划 ID/ROI/消耗/出价明细，Analysis 包含逐条异常标注（🔴 严重 / 🟡 关注）与占比统计，Strategy 包含逐条优化方案与预估日节省金额。超过 3 行自动折叠+渐变遮罩，LLM 节点精确标记，running ◉ 转圈 → done ✓ 绿勾 → failed ✕ 红色完整状态流转。

**产品打磨**：React 18 + TypeScript + Tailwind CSS + shadcn/ui 三栏驾驶舱——左侧对话面板（新手快捷提示 + 受控参数面板 + 确认卡片），中间 Agent 编排追踪（多行输出 + 展开/收起），右侧数据洞察仪表盘（before/after 对比 + 骨架屏 + 更新时间戳）。API 失败不显示空白审批卡，后端断连显示友好错误而非命令行，切换标签时审批流保护弹框，取消操作可撤销。

**容错与安全**：意图识别采用 DeepSeek LLM（3 次重试） + Keyword Fallback 两级机制，Fallback 逻辑用 10 个 pytest 用例覆盖全部 6 个业务场景。后端 FastAPI 通过 ASGI 中间件实现可开关的 API Key 认证，前端所有 fetch 调用统一携带 X-API-Key 头，生产环境一键开启，本地开发自动跳过。

**部署就绪**：前端 Vercel 一键部署（vercel.json 已配置 SPA rewrites + CDN 缓存策略），后端 Railway / Render 部署（uvicorn + 环境变量注入），Docker Compose 本地一键启动（frontend + backend + redis + chroma）。前后端 .env.example 模板完整。

**AI Coding 闭环**：95%+ 代码由 Claude Code 生成，完整践行 Spec → Plan → Tasks → Implement → PM Audit → Iterate 的开发范式。经过资深产品经理视角的 20 项 UX 审计，从安全漏洞到体验缺陷全部修复。
